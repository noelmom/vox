How it works

Three steps from text to audio

Record or upload a short voice clip, then generate studio-quality audio from any text in seconds.

1
🎙

Add a voice profile

Record directly in the browser or drop in an audio file. iPhone Voice Memos work great. Vox converts and stores it locally.

2
✍️

Write your script

Paste any text into the UI or send via API. Choose a built-in tone or open the Custom panel to dial in every parameter exactly how you want it.

3
🎵

Download your audio

Vox generates MP3 or WAV using Apple Silicon acceleration. Play it in the browser, download it, or pipe it into your workflow.

Features

Everything you need, nothing you don't

Built for developers, creators, and teams that need reliable local TTS without the cloud dependency.

⚡️
Apple Silicon acceleration
MPS backend delivers fast inference on M1–M4 chips. Falls back to CPU automatically on any Mac.
🎤
Voice profiles & tags
Store named voices with per-profile TTS defaults. Tag them, filter by tag, and edit settings without re-uploading.
🎛
Tones & custom parameters
Four built-in tones plus a Custom panel — dial in exaggeration, CFG weight, temperature, and more. Saved between sessions.
🎙
In-browser recording
Capture your voice directly in the browser. Live waveform, playback preview, and one-click upload — no external app needed.
📁
Multi-format ingest
Upload WAV, M4A, MP3, AIFF, FLAC, or OGG — or drop files into input/ for auto-registration. Everything converts to WAV.
📊
Live progress & history
Generation ETA bar, real upload progress, and a full job history with timing, RTF, and per-request tracing.
⌨️
Clean REST API
Generate audio with a single curl call. Every response includes an X-Request-ID for easy tracing and debugging.
🖥
Web UI included
Full browser interface for generating, managing voices, and browsing history — no separate frontend server required.
📴
100% offline
No account, no cloud sync, no telemetry. Pull the ethernet cable — Vox still works. Air-gap friendly by design.

API

Simple enough for a curl command

One endpoint, form data, audio back. The response headers tell you everything you need to debug.

Terminal
# generate audio with a named voice profile
curl -X POST http://localhost:8000/tts \
  -F "text=Hello, this is Vox." \
  -F "voice_name=noelmo-normal" \
  -F "preset=youtube" \
  --output audio.mp3

# every response carries timing + tracing headers
X-Request-ID: 149e08c1-7b76-4fc3-88c3...
X-Audio-Duration-Seconds: 3.37
X-Generation-Seconds: 4.12
X-RTF: 1.22

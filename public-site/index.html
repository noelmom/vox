Why Vox

Local voice generation without cloud ceremony.

Vox is built for creators, developers, and automation-heavy workflows that need speech output without uploading every script to a hosted service.

P

Private by default

Your scripts, voice profiles, and generated audio live on your Mac. Vox runs as a local service with no account system or telemetry pipeline.

M

Made for Mac

The one-click package installs the native helper, local server, demo voice profile, and runtime files under standard macOS locations.

A

Agent-ready API

After install, AI agents and scripts can submit text, choose a voice profile, poll async jobs, and download generated MP3 or WAV output.

For AI agents

Give agents a local voice tool.

Vox exposes a local REST API at http://localhost:8000 after installation. That makes it simple for Codex, local automations, shell scripts, and other agents to create speech clips without a cloud TTS account.

Read the agent skill View API source 
curl -X POST http://localhost:8000/api/v1/tts \
  -F "text=Hello from a local AI agent." \
  -F "voice_name=noelmo-normal" \
  -F "preset=default"

# Poll the returned request_id:
curl http://localhost:8000/api/v1/jobs/{request_id}

# Download completed audio:
curl -L http://localhost:8000/api/v1/jobs/{request_id}/audio \
  --output voice.mp3
FAQ

Good questions before you install.

Does Vox run locally?

Yes. Vox runs a local server and web studio on your Mac. The app opens from localhost after install.

Can AI agents use Vox?

Yes. Agents can call the local REST API to generate audio, poll job status, and download MP3 or WAV files.

Does Vox require Homebrew?

Yes for the current one-click installer path. Homebrew provides system dependencies such as Python 3.11 and ffmpeg.

Where are API docs?

After installing and starting Vox, open http://localhost:8000/docs on the same Mac.
